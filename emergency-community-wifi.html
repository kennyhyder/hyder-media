Emergency Community
Internet — Clubhouse WiFi

Prepared for: Community Emergency Preparedness Connection: Community Starlink Coverage: Clubhouse & common areas Recurring software cost: $0

A simple, fair way for residents to share the community Starlink connection at the clubhouse during an emergency — so everyone gets a turn, no one monopolizes the bandwidth, and the system runs itself. Built on standard, off-the-shelf networking gear with no monthly software fees and full remote management, so no on-site IT person is needed once it's installed.

1 How it works

The Starlink connects to a small network gateway that broadcasts WiFi across the clubhouse through ceiling-mounted access points. Residents join the network and sign in with a personal access code handed out in advance. The gateway runs a login page ("captive portal") and automatically enforces fair-use rules — there's nothing for a volunteer to police by hand. The whole system is configured and maintained remotely.

2 Fair-use controls (automatic)

  • Personal login per resident — access codes generated from the resident roster and distributed before an emergency.
  • Data cap per session — the connection ends automatically once a user hits their allotment, freeing the slot for the next person.
  • Time limit per session — enforced only when others are waiting, so no one is bumped off when there's room to spare.
  • Speed cap per user — each person gets a fair, equal slice so a single user can't swamp the connection.
  • Streaming & large downloads blocked — emergency communications only (messaging, email, news, maps), which protects the shared data plan.
  • Concurrent-user cap with a fair queue — when the network is full, new users see a "you're next in line" page and are admitted in order as slots open.
  • First-come, first-served — no assigned time slots or schedules to argue over.
  • Optional priority lane — emergency/medical responders can be exempted from the queue.

4 Starlink plan

The existing Local Priority 500 GB plan is well-suited to episodic emergency use — roughly 30+ hours of a packed, 150-person room doing messaging, email and maps (with streaming blocked). Because the system activates only during emergencies, the most economical approach is to keep the base service active (~$30/mo) so the dish stays ready, and purchase a data bucket only in the month an emergency occurs.

Plan note Starlink discontinued the free "$0 pause" in 2025 — a paused account is now ~$5/month ("Standby Mode"), or it can be fully cancelled for $0 and reactivated when needed. Worth confirming the exact standby terms with Starlink for budgeting.

5 Installation & operation

Physical install (on-site, ~half a day)

  • Mount the gateway & switch near the Starlink.
  • Run network cable to 3 access-point locations & mount them.
  • Connect Starlink and power on.

A local low-voltage/network installer or a handy volunteer can do this from pre-configured, ship-ready equipment.

Configuration (100% remote)

  • Login portal, fair-use rules & resident codes.
  • Network security & guest isolation.
  • Ongoing tweaks & "turn it on" for an event.

All software is set up and maintained remotely via the manufacturer's free cloud — no recurring cost, no on-site IT.

Important — backup power This is an emergency system, so it must keep running when the grid is down. The Starlink and networking gear need backup power — a battery/UPS at minimum, ideally tied to a generator or battery the clubhouse already has. Without it, the system goes dark exactly when it's needed most.
